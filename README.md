# KosWatt

**Smart Energy Agentic AI for Boarding House (Kos) Environments**

KosWatt is an autonomous energy management system that detects wasteful electricity usage in real time and executes corrective actions without human intervention. It combines Fuzzy Logic for handling environmental uncertainty with STRIPS Planning for structured autonomous action sequencing.

---

## How It Works

1. **Sensor inputs** (occupancy, temperature, time of day, device states) are fed to the fuzzy inference engine.
2. **Fuzzy Logic** (Mamdani) computes a continuous waste score (0-100), incorporating ethical guardrails for adaptive devices (AC, lamp).
3. If the score exceeds 65 (HIGH WASTE), the **STRIPS Planner** performs BFS forward search to find the shortest action sequence that brings the system to an energy-efficient goal state.
4. The Streamlit dashboard visualizes the before/after comparison in real time.

## Quickstart

### 1. Clone the repository

```bash
git clone https://github.com/<your-username>/koswatt.git
cd koswatt
```

### 2. Create a virtual environment

```bash
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Run the dashboard

```bash
streamlit run app.py
```

Open `http://localhost:8501` in your browser.

### 5. Explore the notebook (optional)

```bash
jupyter notebook koswatt_model_revised.ipynb
```

## Fuzzy Rule Summary

| Scenario                               | Device Category | Waste Score  | Rationale                          |
|----------------------------------------|-----------------|--------------|-------------------------------------|
| Empty room + TV on                     | Rigid           | HIGH         | No tolerance for non-critical device|
| Empty room + AC on + temp >= 32C       | Adaptive        | MEDIUM       | Thermal comfort guardrail           |
| Empty room + AC on + temp < 30C        | Adaptive        | HIGH         | No thermal justification            |
| Empty room + lamp on + nighttime       | Adaptive        | MEDIUM       | Safety/deterrence tolerance         |
| Empty room + lamp on + daytime         | Adaptive        | HIGH         | Natural light available             |
| Occupied room + high watt              | Any             | HIGH         | Overuse during occupancy            |
| Occupied room + normal watt            | Any             | LOW          | Normal usage                        |

## STRIPS Planner

The planner models device control as a state-space search problem:

- **State**: `{ac_on, lamp_on, tv_on, ac_eco}`
- **Operators**: `TURN_OFF_TV`, `TURN_OFF_LAMP`, `TURN_OFF_AC`, `SET_AC_TO_ECO`
- **Goal**: Derived from fuzzy guardrails (e.g., AC ECO if hot, lamp off if daytime)
- **Search**: BFS forward-chaining, returns shortest action sequence
